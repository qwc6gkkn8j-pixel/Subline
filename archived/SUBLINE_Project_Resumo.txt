## O QUE É O SUBLINE

Plataforma de gestão de barbearias com 3 roles:
- Admin       → controlo total da plataforma
- Barbeiro    → gere os seus clientes e subscrições
- Cliente     → vê a sua subscrição e marca cortes

---

## STACK TÉCNICO ESCOLHIDO

Frontend:  React 18 + TypeScript + Tailwind CSS
Backend:   Node.js + Express + TypeScript
Database:  PostgreSQL + Prisma ORM
Pagamentos: Easypay (gateway PT — MBWay, Multibanco, Cartão)
Chat:      Socket.io (real-time)
Calendário: FullCalendar
Emails:    Nodemailer (Gmail SMTP)
Auth:      JWT (access 1h + refresh 7d) + bcrypt

## FLUXO DE PAGAMENTO (EASYPAY)

1. Admin cria plano na BD (nome, preço, cortes/mês, barbeiro)
2. Barbeiro seleciona cliente + plano → clica "Gerar Link"
3. Backend chama POST https://api.sandbox.easypay.pt/2.0/checkout
4. Retorna URL de pagamento → barbeiro copia e envia ao cliente
5. Cliente paga via MBWay/Multibanco/Cartão na página Easypay
6. Easypay envia webhook → POST /api/webhooks/easypay?secret=xxx
7. Backend ativa subscrição + reset cortes + notifica cliente

## EASYPAY — CHAMADA PRINCIPAL (criar link de pagamento)

POST https://api.sandbox.easypay.pt/2.0/checkout
Headers:
  AccountId: <EASYPAY_ACCOUNT_ID>
  ApiKey:    <EASYPAY_API_KEY>
  Content-Type: application/json

Body:
{
  "type": "sale",
  "payment": {
    "methods": ["mbway", "mb", "cc"],
    "type": "frequent",
    "frequency": "1M",
    "value": 49.99,
    "currency": "EUR"
  },
  "customer": {
    "name": "João Silva",
    "email": "joao@email.com",
    "phone": "912345678"
  },
  "capture": {
    "descriptive": "Plano Gold — SUBLINE"
  }
}

Resposta: { "id": "checkout_xxx", "payment": { "url": "https://pay.easypay.pt/..." } }
→ Guarda o id como easypayCheckoutId na Subscription
→ Envia o url ao cliente

## WEBHOOK EASYPAY — LÓGICA

POST /api/webhooks/easypay?secret=WEBHOOK_SECRET

Pagamento confirmado:
  event.type === "capture" && event.status === "success"
  → subscription.status = "active"
  → subscription.cutsUsed = 0
  → subscription.currentPeriodStart = hoje
  → subscription.currentPeriodEnd = hoje + 30 dias
  → criar Payment (status: "paid")
  → enviar email ao cliente
  → criar Notification

Pagamento falhado:
  → subscription.status = "payment_failed"
  → notificar cliente + barbeiro

Responder sempre: 200 { received: true }

## SISTEMA DE CORTES

- Cada plano tem cutsPerMonth (ex: Gold = 4 cortes/mês)
- Subscription guarda cutsUsed (começa em 0, reset no webhook de renovação)
- Barbeiro regista corte: POST /api/barber/clients/:id/register-cut
  → cutsUsed += 1
  → se cutsUsed >= cutsTotal → notificação "limite atingido"
- Cliente vê progress bar: "3 de 4 cortes usados — renova em 5 dias"

## CALENDÁRIO — LÓGICA DE SLOTS

GET /api/calendar/slots?barberId=xxx&date=2025-06-10
1. Busca horário do barbeiro para esse dia da semana (BarberAvailability)
2. Busca marcações existentes nessa data (Appointment)
3. Busca datas indisponíveis (BarberUnavailable)
4. Calcula slots livres
5. Retorna array de slots disponíveis

## NOTAS IMPORTANTES

- Easypay não tem SDK Node.js oficial → usa axios para chamadas REST
- Webhook não tem assinatura (como Stripe) → protege com ?secret= no URL
- Cada barbeiro NÃO tem conta Easypay própria → só UMA conta (a tua)
- Admin gere os planos, barbeiro gera os links, Easypay processa tudo
- Para produção: criar conta real em easypay.pt (requer documentos empresa)
- NUNCA commitar o .env para o GitHub
