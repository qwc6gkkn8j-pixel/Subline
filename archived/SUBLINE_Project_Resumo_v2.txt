## O QUE É O SUBLINE

Plataforma SaaS de gestão de barbearias com 3 roles:
- Admin       → controlo total da plataforma
- Barbeiro    → gere os seus clientes, subscrições e recebimentos
- Cliente     → vê a sua subscrição e marca cortes

Em produção em: https://subline-client-production.up.railway.app

## GATEWAY DE PAGAMENTOS — STRIPE CONNECT (OFICIAL)

### Decisão de arquitetura
Easypay foi avaliado e descartado. O gateway oficial é Stripe Connect com
Express accounts e Embedded Components. Cada barbeiro tem a sua própria
conta Stripe — a plataforma não toca no dinheiro.

### Como funciona
1. Barbeiro clica "Configurar recebimentos" no Perfil
2. Backend cria automaticamente uma conta Stripe Express para ele
3. Onboarding KYC feito dentro da app (sem sair para stripe.com)
4. Clientes pagam diretamente ao barbeiro via Payment Links Stripe
5. Subline cobra fee ao barbeiro (application_fee_amount)

### Fluxo técnico
- POST /api/barber/stripe/create-account  → cria Express account
- POST /api/barber/stripe/account-session → cria Account Session (embedded UI)
- POST /api/barber/stripe/verify          → verifica charges_enabled
- GET  /api/barber/stripe/status          → estado da conta do barbeiro
- Frontend: @stripe/connect-js + @stripe/react-connect-js

### Env vars necessárias (Railway subline-server)
STRIPE_SECRET_KEY=sk_live_...
STRIPE_PUBLISHABLE_KEY=pk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...
(STRIPE_CLIENT_ID já não é necessário — OAuth removido)

## NOTAS IMPORTANTES

- OAuth Stripe (Standard accounts) foi removido — usar apenas Embedded Components
- STRIPE_CLIENT_ID não é necessário na nova arquitetura
- Cada barbeiro tem a sua própria conta Express → recebe diretamente
- A plataforma usa application_fee_amount para cobrar fee aos barbeiros
- Webhooks idempotentes: verificam stripePaymentIntentId / stripeInvoiceId antes de criar Payment
- O client usa vite preview para servir o SPA no Railway
