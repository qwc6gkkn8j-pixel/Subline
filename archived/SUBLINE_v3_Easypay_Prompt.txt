FLUXO DE SUBSCRIÇÃO (o que implementar)
--------------------------------------------------------------------------------

PASSO 1 — Admin cria plano na BD (sem chamada à Easypay ainda)
  Guardar: nome, preço, cortes/mês, descrição, barbeiro associado

PASSO 2 — Barbeiro/Admin gera link de pagamento para cliente
  POST https://api.sandbox.easypay.pt/2.0/subscription
  Body:
  {
    "method": "mbway",           // ou "mb", "cc"
    "currency": "EUR",
    "customer": {
      "name": "João Silva",
      "email": "joao@email.com",
      "phone": "912345678"       // obrigatório para MBWay
    },
    "value": 49.99,
    "frequency": "1M",           // 1M = mensal, 1W = semanal, 1Y = anual
    "capture": {
      "descriptive": "Plano Gold SUBLINE"
    }
  }
  
  Resposta:
  {
    "id": "sub_xxxx",
    "method": { "mbway": { "alias": "912345678" } },
    "status": "active"
  }
  
  Guardar na BD: easypay_subscription_id

PASSO 3 — Webhook (Easypay notifica quando pago)
  POST /api/webhooks/easypay
  Easypay envia notification quando:
  - pagamento confirmado (type: "capture")
  - pagamento falhado   (type: "generic")
  
  Body do webhook:
  {
    "id": "notif_xxxx",
    "type": "capture",
    "status": "success",
    "messages": ["capture"],
    "date": "2025-06-10 14:00:00",
    "transaction": { "id": "txn_xxxx" }
  }

PASSO 4 — App atualiza subscrição
  Ao receber webhook type="capture" status="success":
  → subscription.status = "active"
  → subscription.currentPeriodStart = today
  → subscription.currentPeriodEnd = today + 30 days
  → subscription.cutsUsed = 0 (reset mensal)
  → send email confirmação ao cliente

PASSO 5 — Pagamento falhado
  Ao receber webhook type="generic" (ou falha):
  → subscription.status = "payment_failed"
  → send email alerta ao cliente
  → notify barbeiro

import axios from 'axios';

const EASYPAY_BASE = process.env.EASYPAY_ENV === 'prod'
  ? 'https://api.prod.easypay.pt/2.0'
  : 'https://api.sandbox.easypay.pt/2.0';

const headers = {
  AccountId: process.env.EASYPAY_ACCOUNT_ID,
  ApiKey:    process.env.EASYPAY_API_KEY,
  'Content-Type': 'application/json',
};

// Gerar link de checkout (para barbeiro enviar ao cliente)
export const createCheckoutLink = async (params: {
  clientName:  string;
  clientEmail: string;
  clientPhone: string;  // obrigatório para MBWay
  amount:      number;
  planName:    string;
  method:      'mbway' | 'mb' | 'cc';
  frequency:   '1M';   // mensal
}) => {
  const response = await axios.post(`${EASYPAY_BASE}/checkout`, {
    type: 'sale',
    payment: {
      methods: [params.method, 'cc'],
      type: 'frequent',
      frequency: params.frequency,
      value: params.amount,
      currency: 'EUR',
    },
    customer: {
      name:  params.clientName,
      email: params.clientEmail,
      phone: params.clientPhone,
    },
    capture: {
      descriptive: `${params.planName} - SUBLINE`,
    },
  }, { headers });

  return {
    checkoutId:  response.data.id,
    paymentUrl:  response.data.payment?.url,
    expiresAt:   response.data.expires_at,
  };
};

// Cancelar subscrição (quando cliente pede cancelamento)
export const cancelSubscription = async (easypaySubId: string) => {
  await axios.delete(`${EASYPAY_BASE}/subscription/${easypaySubId}`, { headers });
};

// Verificar status de pagamento
export const getPaymentStatus = async (paymentId: string) => {
  const res = await axios.get(`${EASYPAY_BASE}/payment/${paymentId}`, { headers });
  return res.data;
};

WEBHOOK HANDLER: POST /api/webhooks/easypay

import { Request, Response } from 'express';
import { prisma } from '../db';
import { sendEmail } from '../services/email.service';

export const easypayWebhook = async (req: Request, res: Response) => {
  // Easypay não usa signature verification como Stripe
  // Valida pelo IP ou segredo no URL: /api/webhooks/easypay?secret=xxx
  const secret = req.query.secret;
  if (secret !== process.env.WEBHOOK_SECRET) {
    return res.status(403).json({ error: 'Unauthorized' });
  }

  const event = req.body;

  // Pagamento confirmado
  if (event.type === 'capture' && event.status === 'success') {
    const checkoutId = event.id; // corresponde ao checkoutId guardado na BD

    const subscription = await prisma.subscription.findFirst({
      where: { easypayCheckoutId: checkoutId },
      include: { client: true, plan: true },
    });

    if (subscription) {
      const now = new Date();
      const nextMonth = new Date(now);
      nextMonth.setMonth(nextMonth.getMonth() + 1);

      await prisma.subscription.update({
        where: { id: subscription.id },
        data: {
          status: 'active',
          cutsUsed: 0,                    // reset mensal
          currentPeriodStart: now,
          currentPeriodEnd: nextMonth,
        },
      });

      await prisma.payment.create({
        data: {
          subscriptionId: subscription.id,
          amount: subscription.plan.price,
          status: 'paid',
          method: event.method || 'unknown',
          easypayTransactionId: event.transaction?.id,
          paymentDate: now,
        },
      });

      // Notificações
      await sendEmail({
        to: subscription.client.email,
        subject: 'Pagamento confirmado — SUBLINE',
        template: 'payment_success',
        data: { plan: subscription.plan.name, amount: subscription.plan.price },
      });

      await prisma.notification.create({
        data: {
          userId: subscription.client.userId,
          type: 'payment_success',
          title: 'Pagamento confirmado',
          body: `O teu plano ${subscription.plan.name} foi renovado com sucesso.`,
        },
      });
    }
  }

  // Pagamento falhado
  if (event.type === 'generic' || event.status === 'failed') {
    // Marcar como payment_failed, notificar cliente e barbeiro
  }

  res.status(200).json({ received: true });
};

CHAT SYSTEM (Real-time com Socket.io)

TABELAS BD:
conversations (id, type ENUM('barber_client','support'), barberId, clientId, adminId, createdAt)
messages      (id, conversationId, senderId, senderRole, content, type ENUM('text','image'), imageUrl, isRead, createdAt)

SOCKET.IO EVENTS:
Client → Server:
  join_room     { conversationId }
  send_message  { conversationId, content, type }
  typing_start  { conversationId }
  typing_stop   { conversationId }
  mark_read     { conversationId }

Server → Client:
  new_message   { message }
  user_typing   { userId }
  user_stopped  { userId }
  message_read  { conversationId }

BACKEND SETUP:
import { Server } from 'socket.io';
const io = new Server(server, { cors: { origin: process.env.FRONTEND_URL } });

io.use((socket, next) => {
  // Verificar JWT no handshake
  const token = socket.handshake.auth.token;
  const user = verifyJWT(token);
  if (!user) return next(new Error('Unauthorized'));
  socket.data.user = user;
  next();
});

io.on('connection', (socket) => {
  socket.on('join_room', ({ conversationId }) => {
    socket.join(`conv_${conversationId}`);
  });
  socket.on('send_message', async ({ conversationId, content, type }) => {
    const msg = await prisma.message.create({
      data: { conversationId, senderId: socket.data.user.id,
              senderRole: socket.data.user.role, content, type }
    });
    io.to(`conv_${conversationId}`).emit('new_message', msg);
  });
  socket.on('typing_start', ({ conversationId }) => {
    socket.to(`conv_${conversationId}`).emit('user_typing', { userId: socket.data.user.id });
  });
  socket.on('typing_stop', ({ conversationId }) => {
    socket.to(`conv_${conversationId}`).emit('user_stopped', { userId: socket.data.user.id });
  });
});

Pede ao Claude Code para gerar o schema.prisma com estes modelos:

User           (id, email, passwordHash, role ENUM(admin,barber,client), status, createdAt)
Barber         (id, userId, name, phone, address, bio, rating, createdAt)
Client         (id, userId, name, email, phone, barberId, createdAt)
Plan           (id, barberId, name, description, price, cutsPerMonth, method, isActive, createdAt)
Subscription   (id, clientId, planId, status, cutsUsed, cutsTotal,
                easypayCheckoutId, currentPeriodStart, currentPeriodEnd, createdAt)
Payment        (id, subscriptionId, amount, status, method,
                easypayTransactionId, paymentDate, createdAt)
Appointment    (id, barberId, clientId, service, date, startTime, endTime,
                durationMinutes, status, notes, clientNotes, cancelledBy, createdAt)
BarberAvailability (id, barberId, dayOfWeek, startTime, endTime, slotDuration, isActive)
BarberUnavailable  (id, barberId, dateFrom, dateTo, reason)
Conversation   (id, type ENUM(barber_client,support), barberId, clientId, adminId, createdAt)
Message        (id, conversationId, senderId, senderRole, content, type, imageUrl, isRead, createdAt)
SupportTicket  (id, requesterId, requesterRole, conversationId, subject,
                category, priority, status, createdAt, resolvedAt)
Notification   (id, userId, type, title, body, isRead, data Json, createdAt)
AuditLog       (id, userId, action, entityType, entityId, details Json, createdAt)

"Build SUBLINE — a full-stack barbershop management platform with:

Tech: React 18 + TypeScript + Tailwind CSS (frontend),
      Node.js + Express + PostgreSQL + Prisma (backend),
      Socket.io (real-time chat), FullCalendar (calendar),
      Easypay REST API (payments — PT gateway, MBWay + Multibanco + card)

Payment flow:
  - Admin creates plans (name, price, cuts/month) stored in DB
  - Barber generates Easypay checkout link per client (POST /2.0/checkout)
  - Client pays via MBWay/Multibanco/card on Easypay hosted page
  - Webhook (POST /api/webhooks/easypay) activates subscription + resets cuts

Features:
  1. Auth: JWT login/register, 3 roles (admin/barber/client)
  2. Admin: user management, plan management, support tickets, audit logs
  3. Barber: client list, cut tracking (cutsUsed/cutsTotal), Easypay link generation, calendar, chat
  4. Client: subscription dashboard, cuts remaining (progress bar), plan change, booking, chat

Design: Primary #0080D0, Accent #8B5CF6, mobile-first with bottom nav,
        SUBLINE logo in all headers, cards with 12px radius and subtle shadows.

Start with: project structure, Prisma schema, auth system, then build each dashboard."
